# CLAUDE.md

## サイト構成

- GitHub Pages の公開元は **`main` ブランチの `/docs` フォルダ**。
- サイトとして配信するファイル（`index.html`, `style.css` など）は `docs/` に置く。
- `README.md` / `CLAUDE.md` などのリポジトリ用ファイルはルートに置く（Pages では配信されない）。

## コミット・PRのルール

- コミットメッセージやPRの説明に、AIが生成した旨（Co-Authored-By など）を**書かない**こと。
- デフォルトブランチ（main）に直接pushしない。必ずブランチを切ってPR経由でマージする。
- PRをマージするときは `gh pr merge --delete-branch` を使う。これ1コマンドでマージ・リモートブランチ削除・ローカルブランチ削除・デフォルトブランチへの切り替えまで行う。
- **PR作成までで一旦止める**。勝手にマージ（＝本番反映）しない。ユーザーがGitHub上でdiffを確認してOKを出してから初めてマージする。
